Core Concept

What is TransactionHeader?

A single, universal table that captures the header-level record of every business event — regardless of department, type, or lifecycle stage.

Technical Definition

TransactionHeader (TH) is the central table in the system where every business interaction is registered. Whether it's a presale quote, a purchase order, an inventory transfer, a customer invoice, or a banking receipt — every single one of them creates exactly one row in TransactionHeader.

Think of it like an envelope: TH stores the "outside of the envelope" — who it's from, who it's to, what type it is, when it was created, what status it's in, and how much it's worth. The detailed contents (line items, products, amounts) live in child tables that link back to this header via the ID field.

The same table serves Presales, Purchases, Inventory, Sales, and Accounting — unified by the TransactionType + TransactionSubType + SpecialType trio which precisely identifies what the transaction is.

How to Read a Transaction
Every TH row is uniquely identified by its ID (backend) and TransactionNumber (frontend). The Type → SubType → SpecialType chain tells you exactly what kind of event it is.
Transaction Lineage
Transactions are linked. A Quote has a ParentTransactionID pointing to the Opportunity. A SO points back to the Quote. An Invoice points to the SO. This chain is tracked via SourceTransactionID to the origin.
Columns Don't All Apply
Not every column is used for every transaction type. Many columns are conditionally populated depending on the TransactionType. See the Field Population Matrix for a visual guide.

It is created the moment a potential customer makes contact — a walk-in, a phone call, an email inquiry — and a salesperson decides the lead is worth tracking. Key takeaways for this step
🔑
Two IDs, two purposes
ID is the backend primary key (used in URLs, joins). TransactionNumber (652118) is the human-facing reference used in all documents, searches, and linking.
📸
Address data is a snapshot
Billing and shipping addresses are copied into TransactionHeader at save time. If the customer's address changes later, this transaction retains the address as it was when the opportunity was created.
🔗
ParentID = 0 means this is the root
An Opportunity has no parent. ParentTransactionID = 0 and SourceTransactionID = 0. Every child transaction (Quote, SO, Invoice) will set these fields pointing back to 652118.
🏷️
SubType is NULL — intentionally
An Opportunity has no SubType. NULL SubType + TransactionType = PreSale is the precise identifier for an Opportunity row. As soon as a Quote is created, a new row is inserted with SubType = "Quote".
💰
Total ≠ Balance at this stage
Total = 2,507 is an estimated value. Balance = 0 because no invoice has been issued yet. Balance becomes meaningful at the Invoice stage when actual payment tracking begins.
🤝
6 associate slots track the influencer network
Party1–Party6 capture everyone who has a hand in bringing the customer: architect, designer, contractor, builder, brand, and referrer. These IDs carry forward into all child transactions for commission and attribution tracking.
